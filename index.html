As we all know JS is a client-side scripting language which means it runs at the client-side inside the browser. But one thing to remember is that clint's browser also needs to support JS otherwise it will not work. The modern website has its own JS engines like Google-chrome has V8, Edge has Chakra, firefox has Spider monkey, etc. The most common use of the JS is validated data which is given by the user in form fields. This way JS works.

The event loop is JS asynchronous programming. We know that JS executes all operations on a single thread but in some data structures, it gives the illusion of multi-threading. It has a runtime model based on an event loop which is used for executing codes, collecting, processing events, and executing queued sub-tasks.

Session storage is kind of local storage. Here the main difference is Session storage data is cleared when the page session end but the local storage data doesn't expired when page session ends and added that when a document is loaded in a tab in browser a unique page session gets created and assigned to that particular tab. Those are the main differences.
